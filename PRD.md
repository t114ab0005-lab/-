# Product Requirements Document (PRD) - CampusShift (時間變現曆)

## 產品是做什麼 ?
CampusShift 是一個針對大學生零碎時間變現的純前端媒合平台 MVP。使用者（學生）可以輸入自己的空檔時間與目標金額，系統會透過 AI（Gemini API）解析需求，並使用 Priority Queue 演算法自動在「不衝堂」的前提下，為使用者排入最高 CP 值的微型任務。同時，任務發布者可以快速在平台上模擬發布即時性的小額懸賞任務。

## 解決誰的問題 ?
主要解決兩類人的問題：
1. **想賺外快的大學生（任務接收方）**：解決他們空堂時間零碎，不想花時間盲目尋找打工職缺，希望「一鍵」將空檔時間有效變現的問題。
2. **急需短期人手的發布者（任務發布方）**：解決店家（如爆滿的麵店）或學生（如需代買午餐的住宿生）臨時缺人，不想走繁瑣傳統招募流程，只需要精準填補當下 1-2 小時勞動力缺口的問題。

## 為什麼值得做 ?
大學生的課表往往充滿 1-2 小時的零碎空堂，傳統打工機制無法彈性配合這種極短期的時間。CampusShift 透過「AI 智能排程」與「優先權佇列演算法」將媒合流程自動化與最佳化，讓空閒時間的價值最大化（高 CP 值優先），大幅降低媒合摩擦力，在校園內創造出一個高效的微型勞動力市場。

## MVP 包含哪些功能 ?
1. **單頁應用 (SPA) 導覽與分頁**：實作流暢的 Navbar 切換，包含「任務大廳」、「AI 智能排程」、「發布任務」、「個人資料」四個區塊，畫面無損沿用 Google Stitch 產出的 UI。
2. **任務大廳 (Mission Hall)**：載入並動態渲染 `jobs.js` 內的 10 筆預設任務清單，展示現有任務狀態。
3. **發布任務模擬 (Publish Mission)**：提供表單讓使用者填寫任務資訊，送出後前端即時將新資料推入 JavaScript 陣列，並立即更新任務大廳畫面（包含空值防禦等前端防呆機制）。
4. **AI 智能排程 (AI Scheduling) - 核心亮點**：
   - 提供行事曆空檔選擇與 AI 指令輸入框（例如：「我想賺 300 元」）。
   - 送出後顯示「AI 運算中」的 Loading 狀態。
   - **意圖解析**：串接 Gemini API，將自然語言指令解析為嚴格的 JSON 格式（若 API 失敗需有降級 Fallback 預設機制）。
   - **核心演算法**：以原生 JavaScript 實作 **Priority Queue (優先權佇列)**，確保排出的任務絕對不與行事曆既有時間重疊（不衝堂驗證），且依據 CP 值由高至低自動排序。
   - **視覺化呈現**：將媒合結果成功排入畫面行事曆，並在瀏覽器 Console 清楚印出 Priority Queue 運作時的排序與比較日誌（Log），以供技術評分驗證。
